Class FileReader (fr)

Class containing all methods required for reading files and writing their content to Storage.storage or save in data. Class requires external library File System (fs) and internal classes Table and Storage.

Method Detail:

fr.readConfigDir(dir)

·         dir <string>: config directory path

Reads directory containing configuration files (columnMapping, ...) and saves their contents in Storage.storage.

Iterates over files in dir by calling fs.readdirSync(<dir>). For each respective file in the dir “Reading file:” + <name of this file> is printed and its content is handled via fr.readAndPutIntoStorage() with the file path as argument.

fr.readAndPutIntoStorage(filePath)

·         filePath <string>: file name or file descriptor

Saves content of file given by filePath into Storage.storage according to specific signal words (PRIORITY,DEFAULT,ATTRIBUTE, none).

Reads content of file described by filePath line by line, except headers (i.e. first line), by calling fr.readByLines(filepath,1). The second column of each line is searched for signal words, i.e. 'PRIORITY', 'DEFAULT', 'ATTRIBUTE' to call respective methods, i.e. fr.handlePriority(), fr.handleDefault(), fr.handleAttribute() with required parameters, i.e. namespace, output path and output value. The different methods specify how the content of the line should be saved in Storage.storage. As lines signed as 'DEFAULT' can also have priority information given in path defining the output value (= content of third column), for those also fr.handlePriority() is called and the adjusted path is used as input for fr.handleDefault(). If no signal word exists in the current line, fr.handlePriority() is called on the respective output describing path (=content of fourth column). For the path defining the location of the input value in data tables (=content of third column), an Array is built in Storage.storage (see Storage.xPathAddToStorage()) containing the corresponding adjusted output paths (=content of fourth column). For all cases a check is performed, if meta information, i.e. user defined transform functions and/or conditions, exists, which is then written to Meta-Storage for the given namespace (Storage.storage.META). 
Note: content of columns in line depend on signal words          

fr.readByLines(path[,skip[,delimiter]])

·         path <string>: filename or file descriptor

·         skip <integer>: number of lines which should be not saved; Default: 0

·         delimiter <string>: separation type of columns in file; Default: ‘\t’

Þ     line-separated content of the path <Array>

Returns content of file described by path as Array of Arrays, with entries of outer Array corresponding to lines in file, except first skip lines of file, and entries of inner Arrays corresponding to columns in line, separated via delimiter.

Reads file content by calling fr.read() on file specified by path. Removes unnecessary spaces and new lines, replaces carriage return with empty string and transforms file content into an Array by splitting it after ‘\n’. The first skip number of entries of the passed Array (i.e. lines of the input file) are optionally shifted and all entries of the Array beginning with ‘//’ (i.e. out commented lines) are removed. All entries of the Array (i.e. lines) are then split by the given delimiter. Returned is an Array (file content) of Arrays (line content).     

fr.read(path)

·         path <string>: filename or file descriptor

Þ     content of the path <string>

Returns content of file described by path 'utf8' encoded by calling fs.readFileSync().

fr.handlePriority(namespace,path)

·         namespace <string>:  identifier of membership of config-information

·         path <string>: defines location of corresponding output value in output format

Þ     path freed from priority-suffix <string>

Saves priority information defined in path into Priority-Storage of given namespace (Storage.storage.PRIORITY[namespace]) and returns path freed from its priority-suffix.

If path has a suffix following the pattern ‘(x)’, with x being an integer indicating the priority of the tag in the output, the suffix is cut from path and the resulting priority information is written into Priority-Storage by calling Storage.xPathAddToStorage(). Returned is the adjusted path.

fr.handleDefault(namespace,path,value)

·         namespace <string>: identifier of membership of config-information

·         path <string>: defines location of corresponding output value in output format

·         value <string>: output value

Saves default information defined in path and value in an Array in Default-Storage of given namespace (Storage.storage.DEFAULT[namespace]).

Converts path into an Array, moves last element of resulting Array as key together with value as value into a Map and reconverts adjusted path to a string. The resulting default information is written into Default-Storage with adjusted path as key by calling Storage.arrayPushToStorage(). Here, method arrayPushToStorage() is used, as whole path is required as key of the newly set Map and not split up into its substrings (compare Storage-methods xPathAddToStorage() and arrayPushToStorage()).

fr.handleAttribute(namespace,path,value)

·         namespace <string>: identifier of membership of config-information

·         path <string>: defines location of attribute in output format

·         value <Object>: {[condition:<Object>,]value:<Object>,path:<string>}

Saves attribute information defined in path and value in an Array in Attribute-Storage of given namespace (Storage.storage.ATTRIBUTE[namespace]) by calling  Storage.xPathPushToStorage().

fr.readDataDir(dir)

·         dir <string>: input data directory path

Þ     contents of dir <Object>

Reads directory containing data files and returns an Object with content of each file as value in form of a table-Class Object and the respective file name as key.

Iterates over files in dir by calling fs.readdirSync(dir). For each respective file in the dir “Reading file:” + name of this file is printed and its content is handled via fr.readData() with the file path as argument. Returned is an Object with file names as keys and file contents as respective values in form of table-Class Objects.

fr.readData(path)

·         path <string>: filename or file descriptor of input data file

Þ     content of path <table-Class Object>

Saves and returns content of file described by path as new table-Class Object with file name (except suffix) as constructor input.

A new Table() with filename without suffix as constructor input is declared. The Content of file described by path is then stored as Array (lines) of Arrays (columns of line separated via ‘$’) in table.data of newly set table-Class Object by calling table.setData(fr.readByLines(path,0,’$’)). The first entry of table.data, which corresponds to the header row in input file, is moved to table.header by calling table.setHeader(table.data.shift()). Returned is the newly built table-Class Object. 

fr.readWordMapping(path)

·         path <string>: filename or file descriptor of file containing wordMapping information

The content of the file described by path is read line by line by calling fr.readByLines(path,1) and written to wordMapping-Storage (Storage.storage.wordMapping) with original data values as keys and transformed values as values (accessible via path defining input value location) by calling xPathAddToStorage().

fr.readFunctions(path)

·         path <string>: filename or file descriptor of file containing user-defined transform functions

Each function-Object given by iteration over fr.read(path) is converted into a function and saved in function-Storage (Storage.storage.functions) in a Map with the name of function as key and the function body as value.

table.deleteMinIndices(keyColumn,versionColumn)

·         keyColumn <string>: name of column containing key values

·         versionColumn <string>: name of column containing version numbers of key values

Removes redundant appearance of same keys at keyColumn position in data by deleting all those entries in data which do not have the maximum value at versionColumn for the given key position by calling table.deleteDataFromIndices(table.getMinIndices(keyColumn,versionColumn)).   

table.getMinIndices(keyColumn,versionColumn)

·         keyColumn <string>: name of column containing key values

·         versionColumn <string>: name of column containing version numbers of key values

Þ     indices corresponding to data with minimum version number <Array>

Returns Array of indices corresponding to data rows with minimum version numbers for keys.

Iterates over entries of field data. If for current entry and respective key at keyColumn position (table.getRevHeader(keyColumn)) no entries exists, the value at versionColumn position (table.getRevHeader(versionColumn)), i.e. key-version, as well as the index of the current entry in data Array, i.e. key-index, is saved as maxIndex for the given key. Otherwise, the current key-version is compared to maxIndex. If current key-version is greater than this stored in maxIndex, the key-index of the old key-version is moved to an Array containing minimum indices and the current key-version together with its key-index is saved as new maxIndex for the given key. If the current key-version is smaller than the old key-version, its key-index is directly moved to the Array containing minimum indices. After the iteration, all entries in the Array containing the minimum indices for a given key are converted to integers and sorted numerically by utils.sortArray(minIndex,numerical). Returned is the Array containing minimum indices.

xml.xPathGetDataFromXML(xPath)

·         xPath <string>: describes location of desired data points

Þ     data accessible via xPath in xml <Array>

Returns data Array accessible via xPath in xml.

As xml is a combination of Maps and Arrays, to simplify iteration, a new Array, returnArr, is initialized containing xml. xPath is converted into an Array by splitting it after ‘/’ and it is iterated over its entries. For each key (i.e. entry of xPath-array), an empty Array, newArray, is initialized and the key is split after ‘[’, to check for iterations. Furthermore, for each key it is iterated over the entries of returnArr. If current entry (type of Map) contains key and the key has an iterator, the respective value of the current key at the iterator-position is pushed to newArray, otherwise, if key has no iterator, the respective value of the current key is attached to newArray. returnArr is set to newArray before iteration of new key starts. Returned is newArray, i.e. all values in xml attached to the last key given by xPath.

·         data <table>: input data (only required for condition check)

·         selectedRow <Array>: data row containing value requiring attribute (only required for condition check)

·         parentPath <string>: path containing iterator information (only required if iterators are extant)

Inserts attributes for current table into xml.

Iterates over attributes stored for table in Storage. For each attribute, iterators are inserted into attribute.path, if parentPath is given, by calling utils.updatePath(parentPath, attribute.path) and, if attribute has conditions, these are checked by calling           utils.resultOfConditionCheck(table,data, parentObj,selectedRow,attribute.condition). If attribute.condition is fulfilled or not extant, attribute.value is written at the required position (attribute.path) in xml in form of a Map ({@ =>attribute.value}).

xml.insertDefault()

Inserts defaults into xml.

Iterates over entries of Default-Storage. If xPath (=key of current entry) is an empty string, each of the respective defaults is added to xml at its requested position by calling xml.XPathSetValue() with key-value pair of current default-Map as input parameters. Otherwise, it is checked if entries for the required xPath exist in xml by calling xml.xPathGetDataFromXML(xPath). If minimum one entry exists ( type of Map), it is iterated over all its entries and all defaults stored for the current xPath are added to the current Map at the right position, given by key of current default-Map, by calling Storage.XPathAddToMarray(insertKey,insertValue,Map).

·         fields <Array>: describes location, where value should be stored in map

·         value <string>: value, which should be stored in map

·         map <Map>: Object hosting data

Iterates over fields and separates each entry into key and iterator, by splitting it after ‘[’. If no iterator exists, iterator is set to 0. When the last entry is reached, value is added to the value-Array of the key at the respective iterator-position in map.

·         map <Map>: required to be sorted

·         path <string>: describes starting point for sorting ;Default: ‘’

Þ     sorted map <Map>

Sorts the given map recursively until the given map is a primitive (utils.isPrimitive(map)).         

It iterates over the keys of the map and, if the key does not signs its value as an attribute (key = @), it iterates over the Array getting from the map.get(key) and sorts the member of the Array by calling itself with the current path+key (utils.mapSorter(map.get(key)[index],path+key)).

utils.tableWalker(data,table,parentObj,xml)

·         data <table>: input data

·         table <string>: name of table to be walked

·         parentObj <Object>: contains dependency-information

·         xml <xml-Class Object>

Adds data values and related attributes to xml by recursively iterating through all rows of table and its subtables containing information about a certain subject.     
 
If current table has iterator information configured in Iterator-Storage (Storage.ITERATOR), it is iterated over all rows of the current table, which have an identifying value (i.e. keyValue) in a specific column (i.e. linkedKey; keyValue as well as linkedKey are predefined in parentObj).  Otherwise only the first matching row is chosen, as the other rows of the current table do not have an additional gain of information for the given subject. For each row, a copy of parentObj (i.e. subParentObj) is updated with iterator information (utils.updateParentObjWithCurrentIterator()), to be able to set the parent Path (utils.createParentIteratorPath()). Parent Path is then used to write important data values of the current row (utils.dataToDataItem()) and to insert attributes (utils.insertAttribute()) at the right position in the xml. Afterwards, it is iterated over all related subtables of the current table and, if the linkage between table and subtable is not already part of subParentObj (utils.checkNewKey()), subParentObj is updated with linking information by calling utils.updateParentObj() and subtable is walked for the current subParentObj by the method calling itself (utils.tableWalker()).

utils.createParentIteratorPath(parentObj)

·         parentObj <Object>: contains iterator information

Þ     path containing iterator information <string>

Inserts for given parentObj iterator values into path also given by parentObj and returns newly built path.

It iterates over all tables which are part of Iterator-Storage (Storage.ITERATOR). For all those tables, which are part of parentObj, their paths, which must be adjusted (parent), and respective iterator values (value) are retrieved from parentObj and compared to parentPath (also considered as child in Code), which corresponds to the already adjusted path from preceding path manipulations (in the first run parentPath is identical to parent). Both are transformed into Arrays and the last parent.length-parentPath.length entries in parent-Array are added to parentPath-Array. In a same step, the value corresponding to the current parent is added in square brackets to the parent.length entry in parentPath-array. Returned is parentPath as path resulting after the last iteration step. Iteration order of tables does not have to be according to dependencies on each other, as the case, that parentPath.length > parent.length is also considered.                

utils.strMapToObj(strMap)

·         strMap <Map|Marray>

Þ     converted strMap <Object>

Converts given strMap recursively to type Object as long as the constructor.name of the given strMap is neither string nor Object.

It firstly initializes a new void Object obj. If type of strMap does not equal ‘object’ (consider Array and Map have return value object for typeof operator), obj is set to strMap. Otherwise, it is iterated over key-value pairs of strMap. If constructor.name of value equals ‘Object’, obj[key] is directly set to value. Otherwise, value is converted to type Object by recursively calling itself (utils.strMapToObj(value)) and saved as value of obj[key]. Hereby, a distinction is made whether original value only host one or multiple entries. For the latter, every single entry of value is converted to type Object by the method calling itself and pushed into an Array, which is then used as value for obj[key]. This distinction is necessary, as strMap may be a combination of Maps and Arrays (see xml.xml). Returned is the in obj saved converted strMap.
